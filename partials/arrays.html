Массивы

Массивы в JavaScript - это упорядоченные коллекции элементов, которые могут быть разных типов. Они являются одним из самых часто используемых типов данных для хранения списков.

Создание массивов

// 1. Литерал массива (предпочтительный способ)
let fruits = ["яблоко", "банан", "апельсин"];

// 2. Конструктор Array
let numbers = new Array(1, 2, 3, 4, 5); // Массив [1, 2, 3, 4, 5]
let arrayWithLength = new Array(5); // Создает массив с 5 ПУСТЫМИ слотами! Не [5]!

// 3. Пустой массив
let emptyArray = [];

// 4. Массив с разными типами данных
let mixedArray = [1, "два", true, { name: "Иван" }, [5, 6]];

// 5. Array.from() - создает массив из итерируемых объектов (например, строк)
let letters = Array.from("привет"); // ["п", "р", "и", "в", "е", "т"]

// 6. Array.of() - создает массив из аргументов (надежнее new Array для одного числа)
let singleNumArray = Array.of(7); // [7]

Доступ и изменение элементов

let fruitsAccess = ["яблоко", "банан", "апельсин", "киви"]; // Используем новое имя

// Доступ по индексу (индексы начинаются с 0)
console.log(fruitsAccess[0]); // "яблоко"
console.log(fruitsAccess[2]); // "апельсин"

// Изменение элемента по индексу
fruitsAccess[1] = "груша";
console.log(fruitsAccess); // ["яблоко", "груша", "апельсин", "киви"]

// Длина массива (количество элементов)
console.log(fruitsAccess.length); // 4

// Доступ к последнему элементу
console.log(fruitsAccess[fruitsAccess.length - 1]); // "киви"

// Доступ к несуществующему индексу
console.log(fruitsAccess[10]); // undefined

Основные методы массивов (Мутирующие и нет)

Существуют методы, которые изменяют (мутируют) исходный массив, и методы, которые возвращают новый массив, не трогая старый.

let fruitsMethods = ["яблоко", "банан"]; // Новое имя
console.log("Исходный:", fruitsMethods);

// --- Методы, ИЗМЕНЯЮЩИЕ исходный массив ---
// push(), pop(), unshift(), shift(), splice(), reverse(), sort()
// (Примеры см. в коде js/02_arrays.js)
console.log("Примеры мутирующих методов в консоли JS...");


// --- Методы, ВОЗВРАЩАЮЩИЕ новый массив (не изменяют исходный) ---
const fruitsOriginal = ["яблоко", "банан", "апельсин"];
console.log("\nИсходный для не мутирующих:", fruitsOriginal);

// concat(): объединяет массивы
let moreFruits = ["киви", "груша"];
let allFruits = fruitsOriginal.concat(moreFruits, ["манго"]);
console.log("concat:", allFruits);
console.log("Исходный после concat:", fruitsOriginal); // Не изменился

// slice(startIndex, endIndex): возвращает часть массива
let slicedFruits = allFruits.slice(1, 4);
console.log("slice(1, 4):", slicedFruits);
console.log("Исходный allFruits после slice:", allFruits); // Не изменился

// join(separator): объединяет элементы в строку
let fruitsString = fruitsOriginal.join(" | ");
console.log("join:", fruitsString);

Методы перебора, трансформации, поиска и сортировки

Эти методы позволяют удобно обрабатывать каждый элемент массива, часто с использованием колбэк-функций. Большинство из них являются чистыми функциями (не изменяют исходный массив).

Метод map()

Используется для трансформации каждого элемента массива. Создает новый массив той же длины.

Аналогия: Конвейер, где каждую деталь красят или изменяют.

const planetsMap = ["Earth", "Mars", "Venus"];
const planetsInUpperCase = planetsMap.map(planet => planet.toUpperCase());
console.log("map upper:", planetsInUpperCase); // ["EARTH", "MARS", "VENUS"]
console.log("Оригинал:", planetsMap); // Не изменился

Практическое задание: Длины названий планет

const planetsForLengthMap = ["Earth", "Mars", "Venus", "Jupiter"];
const planetsLengths = planetsForLengthMap.map(planet => planet.length);
console.log("Длины планет:", planetsLengths); // [5, 4, 5, 7]

Работа с массивами объектов:

const studentsMap = [ { name: "Mango", score: 83 }, { name: "Poly", score: 59 } ];
const names = studentsMap.map(student => student.name);
console.log("Имена студентов:", names); // ["Mango", "Poly"]

Метод flatMap()

Похож на map(), но дополнительно "разглаживает" результат на один уровень. Полезен, когда колбэк возвращает массив.

const studentsWithCoursesFlatMap = [ { name: "Mango", courses: ["math", "phys"] }, { name: "Poly", courses: ["sci", "math"] } ];
const flattenedCourses = studentsWithCoursesFlatMap.flatMap(student => student.courses);
console.log("flatMap courses:", flattenedCourses);

Практическое задание: Список всех жанров книг

const booksWithGenres = [ { title: "Book 1", genres: ["adv", "hist"] }, { title: "Book 2", genres: ["fic"] } ];
const genres = booksWithGenres.flatMap(book => book.genres);
console.log("Все жанры:", genres);

Методы filter() и find()

Используются для поиска элементов по критериям.

Метод `filter()` (Фильтрация)

Отбирает **все элементы**, удовлетворяющие условию. Возвращает **новый массив**.

Аналогия: Сито.

const valuesFilter = [51, -3, 27, 21, -68];
const positiveValues = valuesFilter.filter(value => value >= 0);
console.log("filter (>0):", positiveValues); // [51, 27, 21]
console.log("Оригинал:", valuesFilter); // Не изменился

Практическое задание: Четные и нечетные

const numbersFilter = [17, 24, 82, 61, 36];
const evenNumbers = numbersFilter.filter(number => number % 2 === 0);
const oddNumbers = numbersFilter.filter(number => number % 2 !== 0);
console.log("Четные (filter):", evenNumbers); // [24, 82, 36]
console.log("Нечетные (filter):", oddNumbers); // [17, 61]

Метод `find()` (Поиск Первого)

Ищет **первый элемент**, удовлетворяющий условию. Возвращает **элемент** или `undefined`.

Аналогия: Поиск первого человека в красной шляпе.

const colorOptionsFind = [ { label: "red"}, { label: "blue"}, { label: "green"} ];
const blueOption = colorOptionsFind.find(option => option.label === "blue");
console.log("find(blue):", blueOption); // { label: 'blue' }
const yellowOption = colorOptionsFind.find(option => option.label === "yellow");
console.log("find(yellow):", yellowOption); // undefined

Практическое задание: Поиск книги

const booksFind = [ { title: "A", author: "X" }, { title: "B", author: "Y" } ];
const BOOK_TITLE_FIND = "A";
const bookWithTitleFind = booksFind.find(book => book.title === BOOK_TITLE_FIND);
console.log("Найдена по названию:", bookWithTitleFind);

Сравнение `filter()` и `find()`

Характеристика filter(callback) find(callback)
Цель Найти ВСЕ элементы Найти ПЕРВЫЙ элемент
Возвращает Новый массив Элемент или undefined
Прерывание Нет Да
Сценарий Подмножество Уникальный элемент

Методы every(), some() и reduce()

Метод `every()` (Все ли?)

Проверяет, удовлетворяют ли **все** элементы условию. Возвращает `true` или `false`.

console.log([1, 2, 3].every(value => value >= 0)); // true
console.log([1, -2, 3].every(value => value >= 0)); // false

Практическое задание: Все ли активны?

const usersEvery = [ { isActive: true }, { isActive: false } ];
const isEveryUserActive = (users) => users.every(user => user.isActive);
console.log("Все активны?", isEveryUserActive(usersEvery)); // false

Метод `some()` (Хотя бы один?)

Проверяет, удовлетворяет ли **хотя бы один** элемент условию. Возвращает `true` или `false`.

console.log([-1, -2, 3].some(value => value >= 0)); // true
console.log([1, 2, 3].some(value => value < 0)); // false

Сравнение `every()` и `some()`

Метод Проверяет... `true`, если... `false`, если... Останавливается?
every() **ВСЕ** Колбэк `true` для всех Колбэк `false` хотя бы раз На первом `false`
some() **ХОТЯ БЫ ОДИН** Колбэк `true` хотя бы раз Колбэк `false` для всех На первом `true`

Практическое задание: Есть ли активные?

const usersSome = [ { isActive: false }, { isActive: true } ];
const isAnyUserActive = (users) => users.some(user => user.isActive);
console.log("Есть активные?", isAnyUserActive(usersSome)); // true

Метод `reduce()` (Сведение)

Последовательно обрабатывает элементы, **накапливая результат** в аккумуляторе. Возвращает одно финальное значение.

Аналогия: Снежный ком.

Синтаксис: reduce((acc, el, idx, arr) => newAcc, initialValue)

initialValue - начальное значение аккумулятора (важно!).

const numbersReduce = [2, 7, 3, 10];
const totalSumReduce = numbersReduce.reduce((sum, number) => sum + number, 0);
console.log("Reduce(сумма):", totalSumReduce); // 22

Практическое задание 1: Общее игровое время

const playtimesReduce = [1270, 468, 710, 244];
const totalPlayTime = playtimesReduce.reduce((total, time) => total + time, 0);
console.log("Общее время:", totalPlayTime); // 2692

Практическое задание 3: Общий баланс

const usersBalanceReduce = [ { balance: 2811 }, { balance: 3821 }];
const calculateTotalBalance = (users) => users.reduce((t, u) => t + u.balance, 0);
console.log("Общий баланс:", calculateTotalBalance(usersBalanceReduce)); // 6632

Используйте a.localeCompare(b) для правильной сортировки строк.

/* ... (код сортировки строк) ... */

Сортируем по значению нужного свойства.

Метод toSorted() и сортировка

Метод toSorted() (появился в 2023 году) создает новый отсортированный список, не меняя оригинал. Это как сделать копию списка и отсортировать ее, оставив старый список нетронутым.

1. Сортировка по умолчанию (как алфавит для слов)

Без аргументов сортирует как строки, например, числа будут как слова.

const array = [5, 2, 9, 1, 5, 6];
const sortedArray = array.toSorted(); // Сортирует как строки
console.log(sortedArray); // [1, 2, 5, 5, 6, 9]
console.log("Оригинал:", array); // [5, 2, 9, 1, 5, 6] (не изменился)

2. Пользовательская сортировка (как выбрать, что важнее)

Можно передать функцию сравнения, например, для чисел: a - b для возрастания, b - a для убывания.

function compareFn(a, b) {
    if (a > b) return 1; // a больше, ставим b вперед
    if (a < b) return -1; // a меньше, ставим a вперед
    return 0; // равны, не меняем
}

const array = [5, 2, 9, 1, 5, 6];
const sortedArray = array.toSorted(compareFn);
console.log(sortedArray); // [1, 2, 5, 5, 6, 9]
console.log("Оригинал:", array); // [5, 2, 9, 1, 5, 6] (не изменился)

3. Сортировка строк (как алфавит для русского языка)

Для строк используем localeCompare(), чтобы правильно сортировать, например, кириллицу.

const array = ['яблоко', 'груша', 'вишня', 'абрикос'];
const sortedArray = array.toSorted((a, b) => a.localeCompare(b)); // Сортируем по алфавиту
console.log(sortedArray); // ['абрикос', 'вишня', 'груша', 'яблоко']
console.log("Оригинал:", array); // ['яблоко', 'груша', 'вишня', 'абрикос'] (не изменился)

4. Сортировка массива объектов (как сортировать по возрасту)

Можно сортировать список людей по возрасту или имени, не меняя оригинал.

const array = [
    { name: 'John', age: 25 },
    { name: 'Alice', age: 30 },
    { name: 'Bob', age: 20 }
];
const sortedArray = array.toSorted((a, b) => a.age - b.age); // Сортируем по возрасту
console.log(sortedArray); // [{ name: 'Bob', age: 20 }, { name: 'John', age: 25 }, { name: 'Alice', age: 30 }]
console.log("Оригинал:", array); // Оригинал не изменился

5. Сравнение toSorted() и sort() (как старый и новый способ)

sort() меняет оригинал, а toSorted() — нет. Это как копировать список перед сортировкой.

const array = ['Характеристика', 'Изменяет оригинал?', 'Да', 'Нет'];

const sortedWithSort = array.sort(); // Меняет оригинал
console.log(sortedWithSort); // ['Да', 'Изменяет оригинал?', 'Нет', 'Характеристика']
console.log("Оригинал после sort:", array); // ['Да', 'Изменяет оригинал?', 'Нет', 'Характеристика'] (изменился)

const sortedWithToSorted = array.toSorted(); // Создает новый, оригинал не меняется
console.log(sortedWithToSorted); // ['Да', 'Изменяет оригинал?', 'Нет', 'Характеристика']
console.log("Оригинал после toSorted:", array); // ['Да', 'Изменяет оригинал?', 'Нет', 'Характеристика'] (не изменился)
Характеристика sort() toSorted()
Изменяет оригинал? Да Нет
Возвращает Ссылку на оригинал Новый массив
Предпочтение Осторожно Рекомендуемый

Цепочки методов (Method Chaining)

Вызов методов последовательно друг за другом.

const studentsChain = [/* ... */];
const namesChain = studentsChain
  .toSorted((a, b) => a.score - b.score) // 1. Сортируем
  .map(student => student.name);          // 2. Извлекаем имена
console.log("Имена (сорт по баллам):", namesChain);

Практическое задание: Имена авторов (цепочка)

const booksChain = [/* ... */];
const MIN_RATING_CHAIN = 8;
const highRatedUniqueAuthors = booksChain
  .filter(book => book.rating > MIN_RATING_CHAIN)
  .map(book => book.author)
  .filter((author, index, array) => array.indexOf(author) === index) // Уникальные
  .toSorted((a, b) => a.localeCompare(b));
console.log(`Авторы (> ${MIN_RATING_CHAIN} рейт., сорт):`, highRatedUniqueAuthors);

Деструктуризация массивов

Удобный синтаксис для извлечения значений из массивов.

// Деструктуризация массива
        let colors = ["красный", "зеленый", "синий"];
        let [red, green, blue] = colors;
        console.log(red);   // "красный"
        console.log(green); // "зеленый"
        console.log(blue);  // "синий"
        
        // Пропуск элементов
        let [first, , third] = colors;
        console.log(first); // "красный"
        console.log(third); // "синий"
        
        // Значения по умолчанию
        let [a, b, c, d = "желтый"] = colors;
        console.log(d); // "желтый"
        
        // Обмен значениями переменных
        let x = 1;
        let y = 2;
        [x, y] = [y, x];
        console.log(x); // 2
        console.log(y); // 1
        
        // Остаточные параметры
        let [head, ...tail] = [1, 2, 3, 4];
        console.log(head); // 1
        console.log(tail); // [2, 3, 4]
